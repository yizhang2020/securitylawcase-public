# Controls -> Evidence Map (ChoicePoint 2006)

## Purpose

This technical appendix maps controls to objective evidence for ChoicePoint 2006, enabling rapid substantiation of implementation and operating effectiveness. It is used by security, compliance, and legal teams to demonstrate what is deployed, how it is monitored, and what records support examiner or litigation requests.

## Hallucinated writing examples

**Scenario:** In an illustrative period following the FTC 2006 data security settlement and findings on fraudulent account onboarding **(time)**, the Senior Lead Security Engineer **(role)** prepares a controls to evidence map **(type)** for Chief Information Security Officer; Compliance Program Owner **(audience)**.

<p><strong>1) Identity-Proofing and Vetting Controls: </strong></p>
<p><strong>Required Control State: </strong>Customer/applicant verification controls are enforced before granting data access.<br>
<strong>Evidence Artifacts: </strong>Verification workflow logs, approval records, exception register.<br>
<strong>Verification Signals: </strong>Rejected fraudulent applications, false-positive rate, review SLA compliance.</p>

<p><strong>2) Access Governance Controls: </strong></p>
<p><strong>Required Control State: </strong>Least-privilege access with periodic recertification and timely revocation.<br>
<strong>Evidence Artifacts: </strong>Role definitions, access review attestations, deprovisioning tickets.<br>
<strong>Verification Signals: </strong>In-scope access review completion rate, stale access count, revocation latency.</p>

<p><strong>3) Monitoring and Investigation Controls: </strong></p>
<p><strong>Required Control State: </strong>Anomaly and fraud monitoring with documented investigation playbooks.<br>
<strong>Evidence Artifacts: </strong>Alert rules, investigation tickets, escalation records.<br>
<strong>Verification Signals: </strong>Alert triage time, investigation closure rate, repeat-incident trend.</p>

<p><strong>4) Governance and Independent Testing: </strong></p>
<p><strong>Required Control State: </strong>Written security program, accountable owners, and independent assessments.<br>
<strong>Evidence Artifacts: </strong>Program governance records, assessment reports, remediation trackers.<br>
<strong>Verification Signals: </strong>Open high-risk findings aging, closure velocity, control-test pass rate.</p>

</div>
